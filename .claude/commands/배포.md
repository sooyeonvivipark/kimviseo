# 배포 명령어

## 실행 단계

### 1단계: 변경된 파일 확인
```bash
git status
git diff --name-only
```

### 2단계: 커밋 메시지 자동 생성
변경된 파일을 분석하여 커밋 메시지 생성:
- `김비서-데이터/` 폴더 변경 감지 → "데이터: [파일명] 업데이트"
- `dashboard.html` 변경 감지 → "UI: 대시보드 업데이트"
- 기타 변경 → "업데이트: [변경 내용 요약]"

### 3단계: GitHub에 푸시
```bash
git add .
git commit -m "[자동생성된 메시지]"
git push origin main
```

### 4단계: 완료 메시지 출력
```
✅ 배포 완료!
- 커밋: [커밋 메시지]
- 브랜치: main
- 원격: origin
```

## 사용 방법

```
사용자: 커밋하고 푸시해줘
Claude: 배포 명령어 실행 → /배포
```

---

## 주요 기능

- ✅ 자동 변경 파일 감지
- ✅ 맞춤형 커밋 메시지 생성
- ✅ 자동 배포 (add → commit → push)
- ✅ 완료 상태 확인

---

## 주의사항

- 변경사항이 없으면 커밋하지 않습니다
- 충돌이 발생하면 사용자에게 알립니다
- 현재 브랜치: `main`
- 원격 저장소: `origin`
